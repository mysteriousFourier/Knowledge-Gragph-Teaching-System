# 后端编排器

`backend/start_all.py` 是项目推荐启动入口，通常由根目录 `start.bat` 调用。它的目标是让用户一次启动完整系统，而不是手动分别打开多个服务。

## 职责

- 读取 `.env`。
- 解析 Python 解释器。
- 检查端口。
- 生成 `frontend/env-config.js`。
- 启动静态前端服务器。
- 启动教育 API。
- 启动维护 API。
- 写入 `.runtime/processes.json`。
- 自动打开主页。

## 服务关系

| 服务 | 默认端口 | 启动后用途 |
| --- | --- | --- |
| Frontend | 3000 | 浏览器访问页面 |
| Education API | 8001 | 教师端和学生端业务接口 |
| Maintenance API | 8002 | 图谱维护接口 |

前端页面不直接读取 `.env`，而是读取启动器生成的 `frontend/env-config.js`。

## 进程记录

启动器会记录各服务 PID。停止脚本依赖这些记录关闭服务，同时会补充检查端口，处理窗口闪退或进程残留的情况。

## 失败处理

常见失败原因：

- Python 解释器路径无效。
- 端口被其他程序占用。
- `.env` 缺少必要配置。
- 依赖包未安装。
- API Key 无效或网络不可用。

启动失败时应先查看终端输出和 `backend/logs/`。不要把日志提交到 Git。

## 修改建议

- 新增服务时，同时更新启动器、停止脚本、`.env.example` 和本文档。
- 新增前端运行时配置时，只写入 `frontend/env-config.js`，不要写死到页面脚本。
- 自动打开浏览器只应打开主页，不应打开教师端或学生端的私有路径。
