# 数据资产

项目的数据分为三类：可提交的结构化材料、运行时生成数据、不得提交的本地资产。

## 可提交材料

适合提交的内容通常放在 `structured/`：

- 章节结构化文本。
- 公式库。
- 表格库。
- 适合公开的元数据。

课程知识原文建议保留英文，避免翻译导致概念偏差。

单端口部署会依赖 `structured/` 进行首次图谱自动同步，因此公开部署前应确认该目录中的材料可以公开分发。

## 公式库

`structured/formula_library.json` 用于把编号展开为原公式。推荐字段包括：

- 公式编号。
- LaTeX 正文。
- 公式说明。
- 所属章节。
- 来源上下文。

系统会识别：

```text
[[FORMULA:6.35]]
[[SEE_FORMULA:6.35]]
Equation 6.35
```

## 运行时数据

运行时数据不提交：

- `.runtime/chapters.json`
- `.runtime/processes.json`
- 题库反馈缓存。
- 图谱数据库。
- 单端口部署的 `APP_DATA_DIR` 内容。
- 日志。
- 本地索引。

这些数据可能包含个人操作、生成结果或本地环境信息。

## 本地资产

不得提交：

- API Key 和凭据。
- 未公开论文 PDF。
- 本地参考说明。
- 模型权重。
- 外部仓库完整副本。
- 个人学习记录。

## 新增数据规则

新增数据文件时，应优先选择可审阅的 JSON、CSV 或 Markdown。二进制数据库、模型和索引只有在确有必要时才考虑提交，并且必须在 README 或数据文档中解释来源、生成方式、许可证和更新流程。

提交前应确认：

1. 数据来源是否允许公开。
2. 是否包含个人信息。
3. 是否包含密钥或路径。
4. 是否需要在 `THIRD_PARTY.md` 说明。
5. `.gitignore` 是否覆盖生成文件。

## 公开边界

如果材料来自外部论文、教材或仓库，只能在许可证允许的范围内提交。不能提交未公开论文 PDF、本地草稿、个人标注和临时参考说明。
