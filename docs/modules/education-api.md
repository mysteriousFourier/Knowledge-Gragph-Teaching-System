# 教育 API 模块

教育 API 位于 `backend/education/`，默认端口为 `8001`。它是教师端和学生端的主要业务后端。

## 核心职责

- 读取和维护章节缓存。
- 调用 DeepSeek 生成授课文案。
- 调用 DeepSeek 生成选择题。
- 为教师端和学生端提供问答接口。
- 整合知识图谱和结构化材料作为回答依据。
- 展开公式引用并返回可渲染文本。
- 记录教师对题目和选项的反馈。

## 主要文件

| 文件 | 职责 |
| --- | --- |
| `api_server.py` | FastAPI 路由、章节缓存、题库和反馈接口 |
| `claude_api.py` | 模型调用封装和提示词组织 |
| `lecture_generator.py` | 授课文案生成 |
| `qa_generator.py` | 问答上下文组织 |
| `kg_constraints.py` | 图谱证据、公式库、约束文本 |
| `vector_backend_bridge.py` | 图谱后端桥接 |

## 模型路由

| 任务 | 模型 |
| --- | --- |
| 教师端问答 | `DEEPSEEK_FLASH_MODEL` |
| 学生端问答 | `DEEPSEEK_FLASH_MODEL` |
| 授课文案生成 | `DEEPSEEK_PRO_MODEL` |
| 题库生成 | `DEEPSEEK_PRO_MODEL` |
| 深度材料整理 | `DEEPSEEK_PRO_MODEL` |

模型名和 API Key 都来自 `.env`。前端不保存 API Key。

## 问答约束

图谱和结构化材料是优先证据，而不是强制拒答开关。合理行为是：

- 能从图谱或课程材料回答时，优先引用这些内容。
- 用户追问公式、概念或推导时，允许解释已有材料中的相关内容。
- 材料缺失时说明不确定性，而不是直接拒绝所有问题。
- 原文知识优先保留英文，避免翻译误差。

## 题库生成

题库应满足：

- 一题只考察一个明确知识点。
- 题干要像正常测验题，避免出现内部占位符或章节标题拼接。
- 选项长度可读，不复用同一组选项。
- 问公式时选项应是公式或等价数学表达。
- 问概念时选项应是概念表述。
- 解析要说明为什么正确、为什么干扰项不对。
- 公式引用必须尽量展开为原公式。

生成策略：

- 优先调用 `DEEPSEEK_PRO_MODEL`。
- 深度生成默认不设置读取超时，避免模型仍在生成时应用层提前取消请求并浪费 token。
- 模型不可用或返回质量不足时，后端会基于知识图谱证据和结构化材料生成本地候选题。
- 教师端“继续生成”会保留已点赞题目，并补充新的候选题，不再把已有好题当作坏约束。
- 当前候选题库有上限，避免无限扩张导致页面卡顿。

## 题库反馈

教师端反馈分为两层：

- 题目整体点赞/点踩。
- 单个选项点赞/点踩。

当前实现中，反馈会保存到本地运行时数据，并影响后续生成提示词。它不会自动训练神经网络。若未来要训练模型，需要单独设计数据导出、标注协议、训练脚本、评估集和隐私审查。

选项反馈只代表单个选项质量，不代表整题质量。坏选项确认后会优先调用 `DEEPSEEK_PRO_MODEL` 重写该选项；若模型不可用，则使用本地规则生成替代选项。替代选项会避开当前选项、同题历史坏选项和刚生成过的替代项，降低多个坏选项被重写成相同文本的概率。

## 公式展开

后端会从 `structured/formula_library.json` 读取公式库，并处理：

```text
[[FORMULA:6.35]]
[[SEE_FORMULA:6.35]]
Equation 6.35
```

返回前应尽量替换为包含 LaTeX 的公式文本，避免前端只能显示编号。

## 章节缓存

`.runtime/chapters.json` 保存运行时章节状态、授课文案、题库和反馈。该文件不提交到 Git。

## 错误处理

前端应能区分：

- 未登录或登录过期。
- API Key 缺失。
- 模型调用失败。
- 图谱数据缺失。
- 题库为空或生成失败。

后端错误消息应便于定位，但不能包含密钥。
